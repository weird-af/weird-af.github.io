OSI model

osi consist of 7 layers.

  • Application

    • It works almost exclusively with applications, providing an interface for them to use in order to transmit data.when data is passed to application layer, it pass that to presentation layer.

  • Presentation

    • the data is recieved from application layer and The presentation layer translates the data into a standardised format, as well as handling any encryption, compression or other transformations to the data and the pass it to session.

  • Session

    • the session layer try to set up a connection between other computer through a network,it send an error msg if it cant. the job of the session layer is to maintain the connection that is established & the data is passed to next layer.

  • transport

  • Network

    • The network layer is responsible for locating the destination of your request

  • Data link

    • The data link layer focuses on the physical addressing of the transmission.the job of the data link layer to present the data in a format suitable for transmission.the data link layer check whether the data is corrupted dduring the recieve.

  • Physical

    • this one is near the hardware of computer.here is where the data which transfer over a network are sent & recieved, the main job of this layer is to convert the binary data of the transmission into signals and transmit them across the network.

TCP/IP model

its similiar to OSI model, it serves as the basis for real-world networking, the TCP/IP model is 4 types -

  • Application
  • Transport
  • Internet
  • Network interface

Q & A detailed.

Q. Which model was introduced first, OSI or TCP/IP?

A. TCP/IP
The TCP/IP model is, in many ways, very similar to the OSI model. It’s a few years older, and serves as the basis for real-world networking

Q. Which layer of the TCP/IP model covers the functionality of the Transport layer of the OSI model (Full Name)?

A. Transport

Q. Which layer of the TCP/IP model covers the functionality of the Session layer of the OSI model (Full Name)?

A. Application

Q. The Network Interface layer of the TCP/IP model covers the functionality of two layers in the OSI model. These layers are Data Link, and?.. (Full Name)?

A. Physical

Q. Which layer of the TCP/IP model handles the functionality of the OSI network layer?

A. Internet

Q. What kind of protocol is TCP?

A. Connection-based
TCP is a connection-based protocol.

Q. What is SYN short for?

A. Synchronise
This request contains something called a SYN (short for synchronise) bit, which essentially makes first contact in starting the connection process.

Q. What is the second step of the three way handshake?

A. SYN/ACK

Q. What is the short name for the "Acknowledgement" segment in the three-way handshake?

A. ACK

ping command

The ping command is used when we want to test whether a connection to a remote resource is possible. Usually this will be a website on the internet, but it could also be for a computer on your home network if you want to check if it's configured correctly.

The basic syntax for ping is ping -target-
eg:- ping google.com

which will return if a network connection availablle on google

Networking tools [dig]

dig command

The command dig is a tool for querying DNS nameservers for information about host addresses, mail exchanges, nameservers, and related information.The most typical use of dig is to simply query a single host.

Dig allows us to manually query recursive DNS servers of our choice for information about domains:.use it like this "dig -domain- @-dns-server-ip-"

Q & A detailed

Q. what is DNS stand for

A. Domain Name System

Q. What is the first type of DNS server your computer would query when you search for a domain?

A. Recursive
Assuming the address hasn’t already been found, your computer will then send a request to what’s known as a recursive DNS server. These will automatically be known to the router on your network. Many Internet Service Providers (ISPs) maintain their own recursive servers, but companies such as Google and OpenDNS also control recursive servers

Q. What type of DNS server contains records specific to domain extensions (i.e. .com, .co.uk*, etc)*? Use the long version of the name.

A. Top-Level Domain
These lower level servers are called Top-Level Domain servers. Top-Level Domain (TLD) servers are split up into extensions. So, for example, if you were searching for tryhackme.com your request would be redirected to a TLD server that handled .com domains.

Q. Where is the very first place your computer would look to find the IP address of a domain?

A. Local cache
As mentioned previously, when your computer queries a domain name, it stores the results in its local cache

Q. Research] Google runs two public DNS servers. One of them can be queried with the IP 8.8.8.8, what is the IP address of the other one?

A. 8.8.4.4

Q. If a DNS query has a TTL of 24 hours, what number would the dig query show?

A. 86400
